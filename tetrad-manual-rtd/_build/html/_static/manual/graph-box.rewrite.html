Graph Box

User-facing instructions. For parameter definitions, see the external Parameter Reference.

What it’s for

Create, view, and edit graphs (DAG, SEM graph, time-lag graph, general graph), copy graphs from other boxes, and run graph operations (subgraphs, CPDAG/PAG conversions, layout, paths, edge-frequency summaries).

When to use it

  • Sketch a causal graph to parameterize or simulate.
  • Inspect/clean a graph produced by a search.
  • Drill into subgraphs, paths, or edge-type frequencies after running searches or bootstraps.

Inputs → Outputs

  • Inputs: Optional parent boxes containing a graph (Graph, PM, IM, Estimator, Data, Simulation, Search, Updater, Regression).
  • Outputs: Graphs for downstream boxes (Compare, PM, Data, Simulation, Search, Knowledge).

How to use

  1. Create/open: Add a Graph box; double‑click to choose graph type (General, DAG, SEM graph, Time-lag).
  2. Add variables: Click the variable tool, then click in the canvas. Toggle measured vs latent as needed.
  3. Add edges: Pick an edge type, drag from source to target. Illegal edges (e.g., cycles in DAGs; back-in-time edges in time‑lag) are rejected.
  4. Manipulate: Use menu items for Subgraphs, Paths, CPDAG/PAG conversions, Make edges (un)directed/bidirected, Generate Complete Graph, Extract Structure Model.
  5. Layout & styling: Use Layout tab; copy/paste layout between compatible graphs.
  6. Save/Load: File ▸ Save (XML, text, JSON, R, DOT image); File ▸ Load (XML, text, JSON).

Common tasks / recipes

Display a Markov Blanket

  1. Create a child Graph box from a parent graph.
  2. Choose Display Subgraphs → Select variables → Type: Markov Blankets → Graph It!

Inspect paths between two variables

  1. Graph ▸ Paths.
  2. Choose mode: Adjacencies, Directed paths, Potentially directed paths, Treks.
  3. Set max length and Update.

Generate a CPDAG from a DAG

  1. Create a child Graph box with the DAG as parent.
  2. Choose Generate CPDAG from DAG.

Random graph for simulations

  1. Open a new Graph box; Graph ▸ Random Graph.
  2. Pick type (DAG, MIM, Scale‑free); set node count, max degree, etc.

Caveats & limitations

  • DAGs: Only directed edges; cycles not allowed.
  • SEM graphs: Directed + bidirected edges only (bidirected = correlated errors).
  • Time‑lag graphs: Edges replicate forward in time; no backward‑time edges.
  • Large/dense graphs: Subgraph isolation and Path/Trek views may be slow.

Related boxes

  • Parametric Model → turns a graph into a parameterized model.
  • Instantiated Model → fixes parameter values for simulation/estimation checks.
  • Search → produces graphs from data (then return here to visualize/manipulate).
  • Compare → edgewise/misclassification/intersection comparisons.
  • Knowledge → encode tiers/constraints used by Search.
